# Paintyard - Сайт магазину фарб та будівельних матеріалів

Простий односторінковий сайт для магазину з адмін-панеллю для управління контентом.

## Особливості

- ✅ Односторінковий дизайн з секціями
- ✅ Липка шапка, що зменшується при прокрутці
- ✅ Секція "Про нас" з описом магазину
- ✅ Секція товарів з горизонтальною прокруткою
- ✅ Секція статей з горизонтальною прокруткою
- ✅ Статті відкриваються в новій вкладці
- ✅ Адмін-панель для управління контентом
- ✅ Можливість змінювати кольори фону
- ✅ Зберігання даних в localStorage
- ✅ Адаптивний дизайн

## Структура файлів

- `index.html` - головна сторінка сайту
- `styles.css` - стилі головної сторінки
- `script.js` - логіка головної сторінки
- `article.html` - сторінка перегляду статті
- `admin.html` - адмін-панель
- `admin-styles.css` - стилі адмін-панелі
- `admin-script.js` - логіка адмін-панелі

## Як користуватися

### Перегляд сайту

1. Відкрийте `index.html` в браузері
2. Переглядайте секції сайту
3. Клікайте на статті щоб прочитати їх повністю

### Доступ до адмін-панелі

1. Клікніть на шестерінку (⚙) в правому нижньому куті
2. Введіть пароль: `admin`
3. Тепер ви можете:
   - Змінювати логотип, адресу, телефони
   - Редагувати текст "Про нас"
   - Додавати, редагувати та видаляти товари
   - Додавати, редагувати та видаляти статті
   - Змінювати кольори шапки та основної сторінки

### Додавання зображень

Для додавання зображень можна використовувати:
- URL зображення (наприклад: `https://example.com/image.jpg`)
- Base64 закодоване зображення
- Data URL (наприклад: `data:image/png;base64,iVBORw0KG...`)

## Деплой на GitHub Pages

1. Завантажте всі файли в репозиторій GitHub
2. Перейдіть в Settings → Pages
3. Виберіть джерело: main branch / root
4. Збережіть налаштування
5. Ваш сайт буде доступний за адресою: `https://username.github.io/repository-name/`

## Зміна пароля адмін-панелі

Відкрийте файл `admin-script.js` та змініть значення:

```javascript
const ADMIN_PASSWORD = 'admin';
```

на ваш власний пароль.

## Зберігання даних

Усі дані зберігаються в localStorage браузера. Це означає:
- Дані зберігаються локально на пристрої користувача
- Кожен користувач бачить свої власні дані
- Для синхронізації даних між користувачами потрібен серверний бекенд
